# 核心业务模块规范（spec.md）

## 模块概述

本模块是项目的核心业务模块，包含所有可复用的业务逻辑、数据操作和模型转换，为定时任务和消息消费模块提供业务能力支持。

## 目录结构约束

### 强制目录结构



```
biz/

├── logic/               # 业务逻辑层

│   └── \*.go             # 按业务领域拆分的逻辑文件

├── dal/                 # 数据操作层（完全对齐Kitex标准）

│   ├── gormL/           # MySQL操作

│   │   ├── {db\_name}/   # 按数据库名隔离

│   │   │   ├── repo/    # 数据表Repo实现

│   │   │   └── where/   # 查询条件构造器

│   │   └── init.go      # MySQL连接初始化

│   └── redis/           # Redis操作

│       ├── {db\_name}/   # 按数据库名隔离

│       └── init.go      # Redis连接初始化

└── convert/             # 模型转换层

&#x20;   └── \*.go             # 按业务领域拆分的转换文件
```

### 目录规则



1. 禁止修改 dal 层的目录结构，必须严格遵循 Kitex RPC 的 dal 规范

2. logic 层按业务领域拆分文件，每个文件对应一个业务领域的逻辑

3. convert 层按业务领域拆分文件，每个文件对应一个业务领域的模型转换

## 分层行为规范

### Logic 层



* 职责：实现核心业务逻辑，处理业务规则、流程编排

* 输入：请求参数、上下文

* 输出：业务结果、错误信息

* 约束：


  * 禁止直接操作数据库，必须通过 DAL 层

  * 禁止重复实现可复用逻辑，所有业务逻辑必须收敛于此

  * 必须携带 ctx 上下文，支持链路追踪和超时控制

### DAL 层



* 职责：封装数据库操作，提供数据访问接口

* 约束：


  * 禁止业务逻辑，仅做数据的增删改查

  * 禁止全局单例 Repo，每个请求独立初始化

  * 必须使用 Where 构造器来构建查询条件，禁止硬编码 SQL

  * 分页必须使用 limit+1 的方式，支持判断是否有下一页

  * 分页结构体必须遵循团队标准模板

### Convert 层



* 职责：实现不同数据结构之间的转换，比如 DO 转 PO、PO 转 DO、请求参数转内部结构

* 约束：


  * 禁止任何业务逻辑，仅做字段映射

  * 禁止数据查询或外部调用

  * 转换逻辑必须可复用，禁止重复实现

## 定时任务模块规范

### 目录结构



```
crontab/

├── main.go              # 定时任务启动入口，仅做初始化和启动

└── task/                # 任务实现目录

&#x20;   └── \*.go             # 按任务拆分的文件，每个文件对应一个定时任务
```

### 行为约束



1. 任务模块仅负责任务的注册、调度和触发，核心业务逻辑必须调用 biz.Logic

2. 禁止在 task 目录中实现业务逻辑，禁止直接操作数据库

3. 每个任务必须携带 ctx 上下文，支持超时控制和链路追踪

4. 任务必须有错误处理，失败时要记录日志，支持重试机制

## 消息消费模块规范

### 目录结构



```
consumer/

├── main.go              # 消费服务启动入口，仅做初始化和启动

└── mq/                  # 消费逻辑目录

&#x20;   └── \*.go             # 按消息主题拆分的文件，每个文件对应一个主题的消费逻辑
```

### 行为约束



1. 消费模块仅负责消息的接收、解析、重试和异常处理，核心业务逻辑必须调用 biz.Logic

2. 禁止在 mq 目录中实现业务逻辑，禁止直接操作数据库

3. 消费必须实现幂等性，避免重复消费

4. 消费失败时必须遵循重试规则，失败超过次数后进入死信队列

5. 每个消费逻辑必须携带 ctx 上下文，支持链路追踪

## 验收标准



1. 所有代码必须符合目录结构约束，无违规目录

2. 所有依赖必须符合依赖规则，无跨层依赖

3. 所有业务逻辑必须收敛在 biz 层，crontab 和 consumer 无重复实现

4. 所有数据操作必须通过 biz.dal 层，无直接数据库操作

5. 所有编码必须符合 Golang 编码规范，无 Lint 告警

6. 所有注释必须使用中文，无拼音或混写命名
